  1. declarative views
  2. compositional components
  3. performance

Event Handlers


            let MyComponent = React.createClass({
              getInitialState(){
                return { who: 'world!', unchanged: 'I dont change' };
              },
              render(){
                return (
                  
                    Hello, {this.state.who}
                  
                )
              },
              handleBlur(event){
                let { who } = this.state;
                this.setState({
                  who: (who == 'you!') ? 'world!' : 'you!'
                })
              }
            })
  • Synthetic Event object works the same on all browsers
  • events: onCopy, onKeyDown, onChange, onSubmit, onDblClick, onDragEnd, onTouchStart, onScroll, ...

Fetching Data From Persistence Layer

  • No blocking methods to wait for data fetch
  • Use componentDidMount life-cycle method
    • Called after the Component was mounted into the DOM
  • Fetch data in here (AJAX/Socket Connection) and then setState

Life-Cycle Methods

  • getInitialState, componentWillMount, componentDidMount, componentWillUnmount, componentWillReceiveProps, componentWillUpdate, ...
  • Add Behaviours and work with other libraries e.g. jQuery plugins
  • Can have more than one of each hook per component
    • Use mixins to add multiple behaviours

Life-Cycle Use Cases

  • componentWillMount Set some initial state depending on props
  • componentDidMount Fetch data from server, add event listener
  • componentWillUnmount Clean-up event listener
  • componentWillReceiveProps Update state based on new props
  • shouldComponentUpdate Prevent unnecessary renders
  • componentDidUpdate Operate on DOM after change
