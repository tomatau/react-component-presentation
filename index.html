Up And Running

Why React?

  1. declarative views
  2. compositional components
  3. performance

Simple Render and respond to events Plays nice with stack (can run in node) Works with whatever for data, routing, etc.. Components(concern) not templates(technology) highly cohesive and loose coupling Less state, build a VDOM and diff it, render whole thing Minimal dom updates and batch update

Tooling

Concise Easy to read with balanced close tags Nice for designers to contribute

Changing state

  • setState(state, callback) - similar to Object.assign
  • more sophisticated signatures in docs
  • React.addons.update - immutability helper
  • replaceState - overwrite previous state (discouraged)
  • setProps & replaceProps - discouraged greatly
  • Pure render methods
  • keys on lists of components
  • Class helpers
